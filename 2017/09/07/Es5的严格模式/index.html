Es5的严格模式

  • JavaScript
  • 技术

最近重温了一下es5的严格模式，以下是一些笔记

设立严格模式的目的：

  1. 消除javascript语法的一些不合理、不严谨之处，减少一些怪异行为
  2. 消除代码运行的一些不安全之处，保证代码安全
  3. 提高编译器的效率，增加运行效率
  4. 为未来的JavaScript做好铺垫

    如何调用严格模式

    • 针对脚本文件
      将”use strict”放在脚本文件的第一行
      将“use strict”直接放在脚本文件的第一行，不利于脚本的合并（严格模式和非严格模式的合并）。
      可以将脚本文件放在(function () {})()。
    • 针对函数
      将”use strict”放在函数体的第一行

      严格模式对javascript的语法和行为的改变

    • 全局变量必须提前声明
    • 禁止使用with
    • eval将生成作用域
    • 禁止this指向全局
    • 禁止在函数内遍历调用栈
    • 无法删除变量，只有当configurable设为true的对象属性，才能被删除
    • 显示报错
    • 对只读属性进行赋值
    • 使用getter方法读取的属性进行赋值
    • 禁止扩展的对象添加新属性
    • 删除一个不可删除的属性
    • 对象不能有重名属性
    • 函数不能有重名参数
    • 禁止八进制表示法
    • 不允许对arguments赋值
    • arguments不再追踪参数变化
    • 禁止使用arguments.calllee
    • 函数必须声明在顶层
    • 一些保留字不能用
